# migrare

Escape vendor lock-in from vibe-coding platforms.

migrare detects lock-in signals in projects generated by Lovable, Bolt, and Replit, then applies automated transforms to give you a clean, portable, self-owned codebase.

**Web tool:** https://migrare.creadev.org  
**Source:** https://github.com/dhaupin/migrare  
**License:** MIT

---

## What it does

Upload a ZIP of your Lovable export (or clone from GitHub). migrare scans every file for:

- Proprietary build tooling (`lovable-tagger`, `componentTagger()` in vite.config)
- Hardcoded credentials (Supabase project URLs baked into generated client files)
- Platform env var bleed (`GPT_ENGINEER_*`, `LOVABLE_*` variable names)
- Direct auth coupling (`@supabase/*` imports scattered across components)

Then it applies transforms: removes the build dep, moves credentials to env vars, renames env keys to portable `VITE_*` equivalents. You download a ZIP with only the changed files. Drop them into your repo and you're free.

## Using the web tool

1. Export your project as a ZIP from Lovable (or clone the GitHub repo and zip it)
2. Go to https://migrare.creadev.org/app
3. Drop the ZIP in the left panel
4. Click **scan project** to see what lock-in was found
5. Click **migrate project** to apply transforms and download the result

No account required. Processing runs on Cloudflare's edge — your files are not stored.

## Architecture

```
functions/
  _engine.js          Self-contained edge engine. Zip parser + Lovable scanner/transforms.
                      No Node built-ins. Runs on the Cloudflare Workers runtime.
  api/
    health.js         GET  /api/health
    scan.js           POST /api/scan    { source: { zip, name } } → ScanReport
    migrate.js        POST /api/migrate { source: { zip, name }, dryRun } → MigrationResult

src/                  TypeScript core (CLI path, full engine, all plugin interfaces)
  core/               MigrareEngine, ProjectGraph, LockInScanner, TransformPipeline
  plugins/lovable/    LovablePlugin — reference IPlugin implementation
  adapters/           GitHub ingest, GitHub PR output, runtime adapters
  auth/               OAuth App + GitHub App + local token providers
  cli/                npx migrare wizard

web/                  Vite + React + React Router SPA
  src/
    AppLayout.jsx     Routes (no BrowserRouter — Prestruct requirement)
    App.jsx           BrowserRouter wrapper for client entry
    main.jsx          hydrateRoot + createRoot
    pages/Home.jsx    Marketing lander
    pages/MigrateApp.jsx  Migration tool UI
  scripts/
    inject-brand.js   Prestruct: injects global meta into dist/index.html
    prerender.js      Prestruct: renders routes to static HTML
  ssr.config.js       Prestruct config: routes, meta, JSON-LD
  public/
    _headers          Cloudflare Pages security + cache headers
    _redirects        No SPA fallback (Prestruct prerendering handles routing)
```

## Local development

```bash
# Web UI (proxies /api to :4242)
cd web && npm install && npm run dev

# Full build with SSR prerendering
cd web && npm run build

# Core TypeScript (CLI + engine)
npm install && npm run build:core
```

## Deploying to Cloudflare Pages

Set the build configuration in your Pages project:

| Setting | Value |
|---------|-------|
| Build command | `cd web && npm install && npm run build` |
| Build output directory | `web/dist` |
| Root directory | `/` (repo root) |

The `functions/` directory at repo root is picked up automatically by Cloudflare Pages as serverless functions.

## Plugin system

migrare is plugin-based. Each platform is a separate `IPlugin` implementation:

```typescript
interface IPlugin {
  meta:          PluginMeta
  detect:        (graph: ProjectGraph) => Promise<DetectionResult>
  getScanners:   () => IScanner[]
  getTransforms: () => ITransform[]
  getValidators: () => IValidator[]
}
```

`LovablePlugin` in `src/plugins/lovable/LovablePlugin.ts` is the reference implementation. See `AGENTS.md` for the full architecture guide.
