# Migrare

Escape vendor lock-in from vibe-coding platforms.

Migrare detects lock-in signals in projects generated by Lovable, Bolt.new, Replit, and v0 (Vercel), then applies automated transforms to give you a clean, portable, self-owned codebase.

**Web tool:** https://migrare.creadev.org  
**Source:** https://github.com/dhaupin/migrare  
**License:** MIT

---

## What it does

Upload a ZIP of your Lovable export. Migrare scans every file for:

- Proprietary build tooling (`lovable-tagger`, `componentTagger()` in vite.config)
- Hardcoded credentials (Supabase project URLs baked into generated client files)
- Platform env var bleed (`GPT_ENGINEER_*`, `LOVABLE_*` variable names)
- Direct auth coupling (`@supabase/*` imports scattered across components)

Then it applies transforms: removes the build dep, moves credentials to env vars, renames env keys to portable `VITE_*` equivalents. Download a ZIP with only the changed files. Drop them into your repo and you're free.

## Authentication

For CLI and web UI to access GitHub repos and open PRs:

| Token | Where | Purpose |
|-------|-------|---------|
| `GITHUB_TOKEN` | CLI env | GitHub PAT for all operations |
| `MIGRARE_TOKEN` | CLI env | Override GITHUB_TOKEN |
| `MIGRARE_GITHUB_CLIENT_ID` | Web/OAuth | OAuth App client ID |

### Creating an OAuth App

For the web UI's GitHub connect flow:

1. Go to https://github.com/settings/developers → OAuth Apps → New OAuth App
2. Fill in:

| Field | Value |
|-------|-------|
| Application name | migrare |
| Homepage URL | `https://migrare.creadev.org` |
| Authorization callback URL | `https://migrare.creadev.org/oauth-callback` |

3. Use the **Client ID** in your code (see `MigrateApp.jsx`, `Nav.jsx`)
4. Add a **Client secret** (generate at bottom of OAuth App page)
5. Set as Cloudflare env variable: `MIGRARE_GITHUB_CLIENT_SECRET`
6. The callback route must be added to `ssr.config.js` routes for prerendering

### Required scopes

For full migrare functionality, your GitHub token needs:
- `repo` - read and write repositories
- `read:org` - read organizations

### Rate limits

The server applies rate limits to protect the GitHub API:
- `/api/auth/status`: 30 requests/minute per IP
- `/api/auth/github/token`: 20 requests/minute per IP
- `/api/auth/repos`: 10 requests/minute per IP
- `/api/scan`: 30 requests/minute per IP
- `/api/migrate`: 20 requests/minute per IP

Token validation results are cached for 5 minutes.

### Security

- Tokens are stored in server memory only (never persisted)
- Web UI uses sessionStorage (cleared on tab close)
- All paths are sanitized against directory traversal
- Request bodies limited to 1MB

```
web/
  functions/api/[[route]].js   ← CF Pages Functions (API)
  dist/                        ← build output (static assets)
  src/                         ← React source
  ...
```

Set these in your **CF Pages project settings**:

| Setting | Value |
|---------|-------|
| Root directory | `web` |
| Build command | `npm install && npm run build` |
| Build output directory | `dist` |

CF Pages automatically picks up `web/functions/` as serverless functions when root directory is `web`.

## Architecture

```
web/
  functions/
    api/
      [[route]].js    Self-contained catch-all function.
                      Handles GET /api/health, POST /api/scan, POST /api/migrate.
                      Engine (zip parser + Lovable scanner/transforms) is fully
                      inlined — no cross-file imports, no bundler needed.
  src/
    AppLayout.jsx     Routes (no BrowserRouter — Prestruct requirement)
    App.jsx           BrowserRouter wrapper for client entry
    main.jsx          hydrateRoot + createRoot
    pages/Home.jsx    Marketing lander
    pages/MigrateApp.jsx  Migration tool UI
  scripts/
    inject-brand.js   Prestruct: injects global meta into dist/index.html
    prerender.js      Prestruct: renders routes to static HTML
  ssr.config.js       Prestruct config: routes, meta, JSON-LD
  public/
    _headers          CF Pages security + cache headers (no # inside route blocks)
    _redirects        No SPA fallback (Prestruct prerendering handles routing)
    _routes.json      Tells CF Pages to invoke function for /api/* only

src/                  TypeScript core (CLI + full engine interfaces)
  core/               MigrareEngine, ProjectGraph, LockInScanner, TransformPipeline
  plugins/lovable/    LovablePlugin — reference IPlugin implementation
  adapters/           GitHub ingest, GitHub PR output, runtime adapters
  auth/               OAuth App + GitHub App + local token providers
  cli/                npx migrare wizard
```

## Local development

```bash
# Web UI (proxies /api/* to :4242 via vite proxy)
cd web && npm install && npm run dev

# Full build with SSR prerendering
cd web && npm run build

# Core TypeScript (CLI + engine)
npm install && npm run build:core
```
